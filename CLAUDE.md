## Commands

```bash
pnpm dev              # Start dev server (Next.js + Turbopack)
pnpm build            # Production build
pnpm lint             # Run ESLint (with 8GB heap limit)
pnpm db:generate      # Generate Drizzle migrations from schema changes
pnpm db:migrate       # Apply migrations to Neon/PostgreSQL
pnpm db:studio        # Open Drizzle Studio UI
pnpm terminal:ssh     # Start the SSH terminal resume server
ANALYZE=true pnpm build  # Build with bundle analyzer
```

There are no test scripts in this project.

## Architecture

This is a **Next.js 16 App Router** portfolio site with React 19, TypeScript, and Sass modules.

### Directory structure

```
src/app/              # App Router — pages, layouts, API routes, contexts, components
src/lib/mdx/          # Blog post loader (getBlogBySlug, getAllBlogs, extractHeadings)
content/blog/         # MDX blog posts (each exports a `metadata` object)
db/                   # Drizzle ORM schema (anonymousUsers, blogReactions) + db client
migrations/           # SQL migration files (generated by drizzle-kit)
terminal/             # Standalone SSH server for the interactive terminal resume
public/               # Static assets
```

### Key data flows

**Blog pipeline**: MDX files in `content/blog/` are imported dynamically via `src/lib/mdx/index.ts`. The `metadata` export in each `.mdx` file is the source of truth for title, slug, date, categories, etc. Heading IDs are generated by `rehype-slug` and mirrored by `extractHeadings()` for the table of contents.

**Reactions**: Anonymous emoji reactions are persisted in Neon/PostgreSQL via Drizzle ORM. The `db/schema.ts` defines two tables: `anonymous_users` (browser-fingerprinted) and `blog_reactions`. The API routes live at `src/app/api/reactions/route.ts`.

**Music delivery**: Tracks are stored in AWS S3 and served via CloudFront with signed URLs. The audio player (`src/app/components/AudioPlayer/`) has its own hooks for playback (`useAudioPlayback`), queue management (`useQueueManager`), and waveform visualization (`useVisualizer`).

**Contact form**: `src/app/api/sendMail/route.ts` sends emails via Resend using HTML templates in `src/app/utils/apiUtils/`.

**Theme**: `ThemeContext` stores dark/light preference in `localStorage` and toggles `theme-light` class on `<body>`. Sass variables in `src/app/styles/variables.scss` drive all theming; components check `.theme-light &` for light-mode overrides.

**Terminal resume**: `terminal/server.ts` runs an independent SSH server (using `ssh2`) that renders an interactive resume. Content lives in `terminal/resumeData.ts` and should stay in sync with `public/assets/Akshay_Gupta_CV.pdf`.

### Styling conventions

- Global styles: `src/app/styles/globals.scss` (imports all partials)
- Sass variables: `src/app/styles/variables.scss` (`$px-theme`, `$px-bg-1`, etc.)
- Per-component: CSS Modules (`.module.scss`) co-located in component directories
- No CSS-in-JS — pure Sass throughout

### React Compiler

`reactCompiler: true` is enabled in `next.config.mjs`. The compiler handles memoization automatically; avoid adding manual `useMemo`/`useCallback` unless there is a measurable reason.

### Adding a blog post

Create `content/blog/<slug>.mdx` with a top-level `metadata` export:

```mdx
export const metadata = {
  title: 'Post Title',
  slug: 'post-slug',
  publishedAt: 'YYYY-MM-DD',
  categories: ['tag1'],
  coverImage: '/images/blog/cover.webp',
  coverImageAlt: 'Alt text',
  author: { name: 'Akshay Gupta', avatar: '/images/blog-author.webp' },
  excerpt: 'Short description.',
};
```
